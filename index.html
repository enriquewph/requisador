Lineamientos para Escribir Requisitos Detallados

Los requisitos de comportamiento y rendimiento definen cómo el sistema debe cambiar las variables controladas en respuesta a los cambios en las variables monitoreadas. Esta guía está basada en la práctica 2.8 del handbook de ingeniería de sistemas.

1. Utilice nombres específicos de variables y modos

Use los nombres exactos de las variables monitoreadas y controladas, los modos del sistema y las variables internas al escribir los requisitos detallados.

✅ Correcto:

"En modo Operational, SET LED_Estado = Verde, fijo"

❌ Incorrecto:

"El LED debe estar verde cuando funciona"

2. Especifique modos y condiciones claramente

Para cada requisito, especifique los modos del sistema y las condiciones bajo las cuales se aplicará el requisito, seguido del cambio en la variable afectada.

Estructura recomendada:

"SI [modo del sistema] Y [condición específica] ENTONCES [acción sobre variable]"

Ejemplo:

"SI modo es Operational Mode Y se recibe comando RPDO_Start ENTONCES SET Motor_Speed = 1500 RPM"

3. Asegure completitud de requisitos

Los requisitos detallados deben estar completos: se debe asignar un valor ideal a cada variable controlada para cada estado del sistema.

Regla importante:

Utilice "NO ESPECIFICADO" cuando no exista una asignación significativa para un estado del sistema.

Ejemplo:

"En modo Fault Mode, SET Display_Message = NO ESPECIFICADO"

4. Mantenga consistencia

Los requisitos detallados deben ser consistentes: solo se debe asignar un valor ideal a cada variable controlada para cada estado posible del sistema.

⚠️ Evite conflictos como:

R1: "En modo Operational, SET LED = Verde"

R2: "En modo Operational, SET LED = Azul"

5. Evite duplicaciones

Asegúrese de que no haya dos requisitos detallados que se dupliquen entre sí, especificando el mismo resultado para modos y condiciones superpuestos.

6. Organice por función

Presente los requisitos detallados del sistema en la función que produce la variable que se especifica. Esto permite rastrear cada variable directamente hasta su función principal.

7. Defina latencia y tolerancia

Para cada variable controlada, defina:

  • Latencia aceptable junto con la justificación de su valor
  • Tolerancia aceptable numéricamente

Ejemplo:

Latencia: ≤ 100ms (para cumplir con tiempo de respuesta del operador)

Tolerancia: ± 5% (precisión requerida por el proceso)

8. Variables internas

No defina latencia y tolerancia para variables internas - estos parámetros solo aplican para variables controladas que interactúan con el entorno.

💡 Resumen: Estructura de un Requisito Bien Formado

ID: R[número]
Función: [Nombre específico de la función]
Variable: [Nombre específico de la variable controlada]
Componente: [HMI/ECI/Ambos]
Condición: SI [modo del sistema] Y [condición específica]
Comportamiento: ENTONCES [acción específica sobre la variable]
Latencia: [valor numérico con unidades]
Justificación: [explicación del porqué del requisito]
